# 2026-05-15 SF2 player experiment

- Started additive self-contained WebAudio/SF2 player experiment after abandoning legacy embed playback.
- Created `src/musicpad-sf2.template.html` with a minimal Musicpad editor, MIDI menu, WebAudio scheduler, basic MIDI parser, and a built-in SF2 parser/sampler.
- Created `tools/build-sf2-html.py` to embed `src/musicpad.js` and `src/general-user-GS-v2-0-3.sf2` into `dist/musicpad-sf2.html` without npm.
- Built `dist/musicpad-sf2.html`; output is about 41.7 MiB because it embeds the 31 MiB GeneralUser GS SF2 as base64.
- Verification: confirmed SF2 RIFF lists (`INFO`, `sdta`, `pdta`) and syntax-checked extracted executable scripts from the generated HTML.
- Known limitation: browser playback still needs live testing; the sampler covers common SF2 generators and MIDI events but is not a full FluidSynth replacement.

- Updated SF2 HTML after no-sound report: chunked embedded base64 decoding to avoid browser `atob()` size limits, and added soundfont preset count status after parsing. Rebuilt `dist/musicpad-sf2.html` and syntax-checked extracted scripts.

- Fixed reported WebAudio error by sanitizing SF2-derived gain, sustain, pan, release, and scheduled times before AudioParam calls; rebuilt generated HTML and syntax-checked scripts.

- Fixed second WebAudio non-finite error by initializing generator-set additive fields to zero and sanitizing playback-rate/root-key calculations before AudioParam scheduling. Rebuilt generated HTML and syntax-checked scripts.

- Decision: abandon WebAudio/SF2 embedded player experiment and return to the prior project decision: Musicpad generates/downloads MIDI only; playback remains external. Removed generated/template/tool files for the experiment.
